# HeyMate Requirements (MVP + Compliance)

## 1. 機能要件（MVP）

### 1.1 初回起動
- ウェルカム画面表示
- 登録導線: メール / Google / Apple
- 利用規約同意
- プライバシーポリシー同意
- 年齢確認（18歳以上のみ）

### 1.2 プロフィール登録
- 基本情報入力（ニックネーム、年齢など）
- ホームジム（市レベルまで）
- トレーニング目標・得意部位
- 本人確認（身分証チェック）
- 顔写真登録
- 性別登録（同性マッチング制御用）
- 本人確認ステータス管理（pending / approved）

### 1.3 マッチング（ホーム）
- 候補表示
- スワイプ（Like/Skip）
- 相互Likeでマッチ成立
- マッチ成立通知
- 同性のみ表示
- ブロック機能
- 通報機能
- 不適切ユーザー凍結（運営）

### 1.4 チャット
- マッチ後のみメッセージ送受信
- 通知設定
- ジム詳細公開トグル（双方同意で表示）
- 通報ボタン常設
- メッセージログ保存（調査用途）
- 初期はテキストのみ（画像送信なし）

### 1.5 マイページ
- プロフィール編集（ジム、スケジュール、実績）
- 1週間のトレーニングスケジュール表示
- よく行く時間帯の表示
- 設定（通知、プライバシー）
- 公開範囲設定（ジム詳細表示/非表示）
- アカウント非表示（休止）
- データ削除依頼導線

### 1.6 アカウント管理
- ログアウト
- アカウント削除（完全削除）
- サポート/ヘルプ導線
- お問い合わせフォーム
- 運営情報表示（会社名/連絡先）
- 利用規約/プライバシーポリシーの常時閲覧

## 2. 非機能要件
- セキュリティ: 通信の暗号化、認証トークン安全管理
- 可用性: 通常操作でクラッシュしない安定動作
- パフォーマンス: 主要画面の初回表示を体感3秒以内目標
- 監査性: 通報・凍結・問い合わせ対応履歴を保持
- プライバシー: 個人情報最小取得（住所詳細は取得しない）

## 3. 法令・ストア審査対応要件
- 年齢制限（18歳以上）
- 利用規約/プライバシーポリシー同意の証跡管理
- 本人確認フロー
- 通報/ブロック機能
- データ削除要求対応
- 事業者情報の明示

## 4. 受け入れ基準（抜粋）
- 未同意ユーザーは登録完了できない
- 年齢条件未達は利用不可
- 非マッチ状態でチャット不可
- 通報操作は2タップ以内で実行可能
- アカウント削除で個人データが論理削除状態になる

## 5. MVP外（将来）
- 画像/動画チャット
- 異性マッチング
- ウェアラブル連携
- 高度レコメンド
