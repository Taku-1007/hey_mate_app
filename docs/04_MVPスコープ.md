# HeyMate MVP Scope

## 1. MVPの目的
- 合トレマッチング体験が成立するかを検証する
- 安全性・法令対応込みでストア公開可能な最小構成を実現する

## 2. In Scope（MVPで実装）
1. 初回起動（認証導線 + 規約同意 + 年齢確認）
2. プロフィール登録（基本情報、ジムエリア、目標、本人確認、顔写真）
3. マッチング（スワイプ、マッチ通知、同性表示）
4. チャット（マッチ後のみ、テキストのみ、通報導線）
5. ジム詳細公開トグル（双方同意）
6. マイページ（編集、週間スケジュール、時間帯表示）
7. アカウント管理（ログアウト、削除、ヘルプ）
8. 安全機能（通報、ブロック、凍結オペレーション前提）

## 3. Out of Scope（MVPでは実装しない）
1. 画像/動画送信
2. AIレコメンド
3. 異性マッチング
4. SNS的タイムライン
5. ウェアラブル/外部サービス連携

## 4. リリース判定基準
- 主要機能が一連で動作する（登録→マッチ→チャット）
- 通報/ブロック/削除が利用可能
- 利用規約・プライバシーポリシー常時参照可能
- 重大障害（P0/P1）がゼロ

## 5. KPI（MVP検証）
- 登録完了率
- 本人確認完了率
- マッチ成立率
- マッチ後チャット開始率
- 週次継続率

## 6. リスクと対策
- 不正利用リスク: 本人確認 + 通報/凍結
- 目的逸脱リスク: 同性表示 + 利用規約明示
- 位置情報リスク: 市レベル管理 + 詳細は双方同意時のみ共有
