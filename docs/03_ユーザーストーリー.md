# HeyMate User Stories

## 1. 認証・オンボーディング
1. As a 初回ユーザー, I want to choose email/Google/Apple signup, so that I can start quickly.
2. As a 利用者, I want to confirm terms/privacy and age eligibility, so that the app is safe and compliant.

## 2. プロフィール登録
3. As a トレーニー, I want to register my gym area and training goals, so that I can meet compatible partners.
4. As a 安全性重視ユーザー, I want identity verification and face photo checks, so that impersonation risk is reduced.
5. As a 同目的ユーザー, I want same-gender matching control, so that usage stays aligned with service purpose.

## 3. マッチング
6. As a 忙しい社会人, I want swipe-based matching, so that I can find partners with low effort.
7. As a 継続したいユーザー, I want match notifications, so that I can act when opportunities appear.
8. As a 安心して使いたいユーザー, I want block/report actions, so that I can avoid harmful interactions.

## 4. チャット
9. As a マッチ済みユーザー, I want to chat after matching only, so that I avoid unwanted DMs.
10. As a プライバシー重視ユーザー, I want gym detail sharing only after mutual consent, so that location exposure is controlled.
11. As a 一般ユーザー, I want text-first chat, so that abusive image risks are minimized in MVP.

## 5. マイページ
12. As a 継続志向ユーザー, I want weekly training schedule visibility, so that I can maintain routine.
13. As a 自分に合う相手を探すユーザー, I want to show frequent training time, so that matching quality improves.
14. As a ユーザー, I want to edit my profile/settings anytime, so that my information stays current.

## 6. アカウント管理
15. As a ユーザー, I want to delete my account and request data deletion, so that I can control my personal data.
16. As a 困っているユーザー, I want easy access to support/help, so that issues are resolved quickly.

## 7. ペルソナ別重点ストーリー

### 山東 拓（23）
- As a 上京した社会人, I want to find gym partners nearby, so that I reduce loneliness and train consistently.

### 川門さん（29）
- As a 忙しい中級者, I want lightweight scheduling with matched users, so that I do not skip workouts.

### 笹井さん（26）
- As a 成長志向の初級者, I want to match with experienced trainees, so that I can improve form and training quality.

## 8. 優先度
- Must: 1〜11, 15, 16
- Should: 12〜14
- Could: ペルソナ別拡張機能（メニュー提案、フォームレビュー）
